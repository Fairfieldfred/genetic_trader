# Genetic Trader

A dual-codebase project: **Python backend** (genetic algorithm + backtesting engine) with a **Flutter desktop UI** (configuration + evolution monitoring). The Flutter UI generates a Python `config.py` and launches `evolve.py` as a subprocess, streaming real-time progress.

## Python Backend File Map

### Core Genetic Algorithm
- `config.py` ‚Äî All GA parameters, gene definitions (GENE_DEFINITIONS dict + GENE_ORDER list), macro flags
- `genetic_trader.py` ‚Äî `GeneticTrader` class: chromosome holder, gene accessor via `get_genes()`
- `genetic_ops.py` ‚Äî `crossover()`, `mutate()`, `tournament_select()` ‚Äî reads GENE_DEFINITIONS dynamically
- `population.py` ‚Äî `Population` class: manages list of GeneticTrader instances
- `evolve.py` ‚Äî Main evolution loop: creates population, evaluates fitness, selects/breeds, logs results

### Strategy & Backtesting (Backtrader)
- `bt_strategy.py` ‚Äî Backtrader strategies: `GeneticStrategy`, `PortfolioGeneticStrategy`, `MacroAwarePandasData` (custom data feed with macro lines), `create_strategy_from_trader()` factory function
- `calculate_fitness.py` ‚Äî `FitnessEvaluator` for single-stock backtesting (deprecated, use portfolio)
- `portfolio_fitness.py` ‚Äî `PortfolioFitnessEvaluator`: multi-stock backtesting, loads macro data, merges into stock DFs
- `parallel_fitness.py` ‚Äî `ParallelFitnessEvaluator`: multiprocessing wrapper, `_WorkerFunction` handles pickling
- `benchmark.py` ‚Äî Buy-and-hold benchmark calculations for comparison

### Data Layer
- `data_loader.py` ‚Äî `DataLoader`: loads from spy.db, `load_stock_data()`, `load_macro_data()`, `merge_macro_into_stock()`
- `create_macro_table.py` ‚Äî Creates `macro_indicators` table in spy.db
- `populate_macro_data.py` ‚Äî Fetches macro data from FRED API + yfinance, requires `FRED_API_KEY` env var
- `import_macro_csv.py` ‚Äî CSV fallback for macro data import, `--template` generates sample CSV

### Utilities
- `hyperparameter_optimizer.py` ‚Äî Grid search over GA hyperparameters
- `fitness_weight_optimizer.py` ‚Äî Optimizes fitness function weight distribution
- `main.py` ‚Äî Legacy entry point (use `evolve.py` instead)

### Tests
- `test_evolution.py`, `test_genetic_ops.py`, `test_ma_strategy.py`, `test_portfolio.py`, `test_initial_allocation.py`, `test_ma_portfolio.py`

## Gene System Architecture

Genes are defined dynamically in `config.py`. The GA operators (`genetic_trader.py`, `genetic_ops.py`, `population.py`) read `GENE_DEFINITIONS` and `GENE_ORDER` at runtime. **Adding new genes requires only changes to `config.py`** ‚Äî no modifications to GA operator code.

| 3 | `stop_loss_pct` | 1.0‚Äì10.0 | float | Risk Management |
| 4 | `take_profit_pct` | 2.0‚Äì15.0 | float | Risk Management |
| 5 | `position_size_pct` | 5.0‚Äì25.0 | float | Risk Management |
| 6 | `macro_enabled` | 0‚Äì1 | int | Macro (master switch) |
| 7 | `macro_weight` | 0.0‚Äì1.0 | float | Macro influence strength |

### Macro Gene Design

Macro genes act as **filters and modifiers** on the existing MA crossover strategy (not independent signals). `_compute_macro_context()` in `bt_strategy.py` returns:

```python
{
    'position_scale': 1.0,    # Multiplied with position_size_pct on buys
    'block_buys': False,      # Gate: prevents new buy orders entirely
    'stop_loss_adj': 1.0,     # Multiplied with stop_loss_pct
    'take_profit_adj': 1.0,   # Multiplied with take_profit_pct
}
```

When `macro_enabled == 0`, all values are neutral (no modification). Backwards compatible with old 6-gene chromosomes.

## Key Patterns & Gotchas

1. **Strategy factory pattern:** `create_strategy_from_trader(trader)` in `bt_strategy.py` creates a configured `bt.Strategy` subclass from a trader's genes. Macro genes default to 0 when absent.

2. **Parallel fitness pickling:** `_WorkerFunction` in `parallel_fitness.py` serializes evaluator config (including `macro_df`) for multiprocessing. Any new data added to the evaluator must also be added to `_extract_config()` and the `__call__` reconstruction.

3. **Forward-fill for macro data:** Monthly/quarterly macro indicators are forward-filled to daily frequency to avoid look-ahead bias. `DataLoader.merge_macro_into_stock()` handles this with left-join + ffill.

4. **MacroAwarePandasData:** Custom `bt.feeds.PandasData` subclass in `bt_strategy.py` that adds extra lines (vix_close, yield_curve_slope, fed_funds_rate, cpi_yoy, unemployment_rate). When columns are absent from the DataFrame, lines contain NaN and are safely ignored.

5. **Gene function names:** In `genetic_ops.py`, the mutation function is `mutate()` (not `random_mutation`). The crossover function is `crossover()`.

6. **Single-stock mode deprecated:** `_WorkerFunction.__call__()` in `parallel_fitness.py` raises `ValueError` for single-stock mode. Always use portfolio mode (even with `PORTFOLIO_SIZE=1`).

7. **Config hardcoded path:** `ConfigViewModel.saveConfig()` writes to `/Users/fred/Development/Genetic Trader/config.py` (hardcoded absolute path).
